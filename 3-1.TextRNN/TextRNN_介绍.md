# TextRNN - 文本分类模型介绍

## 一、模型概述

TextRNN（Text Recurrent Neural Network）是一种基于循环神经网络（RNN）的文本分类模型。它利用RNN处理序列数据的能力，可以有效地捕捉文本中的上下文信息和长期依赖关系。该模型广泛应用于文本分类、情感分析、语言建模等自然语言处理任务。

## 二、网络结构

### 2.1 整体架构

TextRNN模型由以下主要组件组成：

```
输入层 → 词向量编码 → RNN层 → 输出层 → 预测结果
```

### 2.2 详细结构说明

#### **输入处理阶段**
- **输入句子**：原始文本句子序列
- **词汇表构建**：将所有词汇进行编号，建立词到索引的映射关系
- **One-hot编码**：将每个词转换为One-hot向量表示

#### **RNN层**
- **模型类型**：基础RNN（可扩展为LSTM或GRU）
- **输入维度**：词汇表大小（n_class）
- **隐层维度**：隐藏单元数（n_hidden）
- **序列长度**：输入句子的单词数（n_step）

RNN处理步骤：
1. 接收输入序列和初始隐状态
2. 逐步处理每个时间步的输入
3. 更新隐状态，传递上下文信息
4. 输出每个时间步的输出和最终隐状态

#### **输出层**
- **特征融合**：取RNN最后一个时间步的输出作为文本的特征表示
- **线性变换**：使用全连接层将隐状态映射到类别空间
- **输出维度**：词汇表大小（n_class）

### 2.3 数据流向

```
输入句子（batch_size, n_step, n_class）
         ↓
    RNN处理序列
         ↓
    输出序列（n_step, batch_size, n_hidden）
         ↓
    取最后时间步（batch_size, n_hidden）
         ↓
    线性变换+偏置
         ↓
    预测结果（batch_size, n_class）
```

## 三、模型参数

| 参数 | 说明 | 作用 |
|------|------|------|
| n_step | RNN步数（序列长度）| 处理的单词个数 |
| n_hidden | 隐层维度 | RNN内部状态的维度 |
| n_class | 类别/词汇表大小 | 输入输出维度 |
| batch_size | 批处理大小 | 同时处理的样本数 |
| learning_rate | 学习率 | 优化器步长 |

## 四、关键技术点

### 4.1 因果语言模型（Causal Language Model）
- 输入：句子的前 n-1 个词
- 输出：预测第 n 个词
- 用途：理解模型对序列信息的理解能力

### 4.2 隐状态传递
- RNN在每个时间步更新隐状态
- 隐状态包含了前序步骤的信息
- 最后的隐状态是整个序列的综合表示

### 4.3 交叉熵损失函数
- 用于多分类任务
- 衡量预测分布与真实标签的差异
- 梯度反向传播用于参数优化

## 五、模型训练过程

### 5.1 数据准备
```
原始句子 → 分词 → 编码 → One-hot向量 → 输入模型
```

### 5.2 训练循环
1. **前向传递**：输入数据到模型，获得预测输出
2. **计算损失**：使用交叉熵函数计算预测损失
3. **反向传播**：计算梯度信息
4. **参数更新**：使用Adam优化器更新模型参数

### 5.3 预测阶段
- 使用训练好的模型
- 输入新的文本序列
- 选择概率最高的类别作为预测结果

## 六、模型优缺点

### 6.1 优点
- ✅ 能够捕捉序列中的长期依赖关系
- ✅ 结构相对简单，易于理解和实现
- ✅ 计算效率高，训练速度快
- ✅ 对于较短文本效果较好

### 6.2 缺点
- ❌ 容易出现梯度消失/爆炸问题
- ❌ 对于很长的序列，性能下降
- ❌ 无法并行处理（顺序依赖）
- ❌ 提取的特征能力有限

## 七、实现要点

### 7.1 超参数设置
- `n_step`：根据文本平均长度设置
- `n_hidden`：隐层维度，通常为50-300
- `learning_rate`：学习率通常为0.001左右
- `epochs`：训练轮数，通常为1000-5000

### 7.2 数据处理
- 词汇表构建：统计文本中出现的所有词汇
- One-hot编码：将词转换为高维稀疏向量
- 批处理：提高训练效率

### 7.3 模型保存和加载
- 保存训练好的模型权重
- 支持模型复用和部署

## 八、应用场景

- **文本分类**：将文章分类到不同的类别
- **情感分析**：判断文本的情感倾向
- **语言建模**：预测序列中的下一个词
- **序列标注**：标注序列中每个位置的标签
- **机器翻译**：作为编码器或解码器
