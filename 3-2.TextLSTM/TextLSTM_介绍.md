# TextLSTM 文本分类模型

## 简介

TextLSTM 是一个基于长短期记忆网络（LSTM）的文本分类模型。它利用 LSTM 的能力来捕捉文本序列中的长期依赖关系，从而对文本进行分类。该模型特别适合处理序列数据，可以学习单词之间的顺序关系。

## 核心思想

- **LSTM 层**：通过循环神经网络的 LSTM 单元逐步处理输入序列，记住长期的上下文信息
- **全连接层**：将 LSTM 最后一个时间步的隐状态映射到分类输出空间
- **序列建模**：将文本视为字符或词的序列，通过 LSTM 学习序列特征

## 网络结构

### 整体架构

```
输入序列 (Batch_size × n_step × n_class)
        ↓
    [转置] → (n_step × Batch_size × n_class)
        ↓
    ╔════════════════════════╗
    ║      LSTM 层          ║
    ║  - 输入维度：n_class   ║
    ║  - 隐状态维度：n_hidden║
    ║  - 单层、单向          ║
    ╚════════════════════════╝
        ↓
    最后时间步输出 (Batch_size × n_hidden)
        ↓
    ╔════════════════════════╗
    ║   全连接层 (W + b)    ║
    ║  - 输入维度：n_hidden  ║
    ║  - 输出维度：n_class   ║
    ╚════════════════════════╝
        ↓
    分类结果 (Batch_size × n_class)
```

### 详细说明

#### 1. **输入层**
- **形状**：`[batch_size, n_step, n_class]`
- 使用 One-hot 编码表示输入序列
- 每个字符被编码为一个长度为 `n_class` 的向量（词表大小）

#### 2. **LSTM 层**
```
参数配置：
├─ input_size = n_class（输入维度，对应字母表大小）
├─ hidden_size = n_hidden（隐状态维度，128）
├─ num_layers = 1（单层）
└─ bidirectional = False（单向）

处理过程：
├─ 初始化隐状态 h₀ 和细胞状态 c₀（全0）
├─ 在 n_step 个时间步上逐个处理输入
├─ 每一步计算隐状态更新和细胞状态更新
└─ 输出：所有时间步的隐状态 outputs
```

**LSTM 单元计算**（在每个时间步 t）：
```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)           # 遗忘门
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)           # 输入门
C̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c)       # 候选细胞状态
C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t             # 细胞状态更新
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)           # 输出门
h_t = o_t ⊙ tanh(C_t)                         # 隐状态输出
```

#### 3. **输出处理**
- **提取最后一步**：从 LSTM 输出中取最后一个时间步的隐状态
- **形状变化**：`[batch_size, n_hidden]`

#### 4. **分类层**
```
线性变换：
output = W · hidden_output + b

参数：
├─ W：权重矩阵 (n_hidden × n_class)，无偏置
└─ b：偏置向量 (n_class,)

输出形状：[batch_size, n_class]
```

## 数据处理流程

### 示例数据
- **输入**：单词序列（如 "make", "need"）
- **目标**：预测最后一个字符（如 "make" 中的 "e"）
- **任务**：给定前 n-1 个字符，预测第 n 个字符

### 批处理流程
```
原始数据：['make', 'need', 'coal', ...]
    ↓
对每个单词：
├─ 输入部分：seq[:-1] → ['m','a','k'], ['n','e','e'], ...
├─ 目标部分：seq[-1] → 'e', 'd', 'l', ...
├─ 转换到索引：利用 word_dict
└─ One-hot 编码：每个索引转为向量
    ↓
输出：
├─ input_batch：[batch_size, n_step, n_class]
└─ target_batch：[batch_size]
```

## 参数详解

| 参数 | 值 | 含义 |
|------|-----|------|
| `n_step` | 3 | LSTM 的时间步数（输入序列长度） |
| `n_hidden` | 128 | LSTM 隐层维度 |
| `n_class` | 26 | 分类类别数（字母表大小） |
| `batch_size` | 10 | 批处理大小 |

## 训练过程

```
1. 初始化模型参数
2. 循环 epoch = 1 到 1000：
   a. 前向传播：input_batch → model()
   b. 计算损失：CrossEntropyLoss(output, target)
   c. 反向传播：loss.backward()
   d. 参数更新：optimizer.step()
   e. 每 100 个 epoch 打印损失
3. 预测：使用训练好的模型对测试数据进行预测
```

## 损失函数

- **交叉熵损失（CrossEntropyLoss）**
- 用于多分类任务，结合了 LogSoftmax 和 NLLLoss
- 公式：$\mathcal{L} = -\sum_{i=1}^{n\_class} y_i \log(\hat{y}_i)$

## 优化器

- **Adam 优化器**
- 学习率：0.001
- 结合了动量法和 RMSprop 的优点，自适应学习率

## 模型优势

1. **捕捉长期依赖**：LSTM 通过遗忘门和输入门机制，能够学习长序列中的依赖关系
2. **解决梯度消失问题**：相比基础 RNN，LSTM 通过细胞状态的直接连接缓解梯度消失
3. **简洁高效**：单层单向结构，计算效率高
4. **易于理解**：网络结构简单明晰

## 使用场景

- 字符级文本分类
- 序列标注任务
- 文本补全预测
- 语言建模
- 短文本分类（如情感分析、意图识别）

## 相关改进方向

1. **双向 LSTM**：使用 Bi-LSTM 同时考虑前向和后向信息
2. **多层堆叠**：增加网络深度以学习更复杂的特征
3. **注意力机制**：为不同的输入位置赋予不同的权重
4. **Embedding 层**：使用预训练词向量替代 One-hot 编码
5. **Dropout 和批归一化**：加入正则化防止过拟合
